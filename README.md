# System Tracker Overlay

**System Tracker Overlay** — это лёгкое прозрачное окно-оверлей для Windows, которое отображает в реальном времени показатели производительности системы: загрузку CPU, RAM, GPU (NVIDIA), а также частоту кадров активного приложения. Окно всегда поверх всех окон и не перехватывает клики («кликсквозь»).

## Возможности

- **Мониторинг CPU, RAM, GPU** (NVIDIA через NVML)
    
- **Отображение активного приложения** (имя процесса)
    
    
    - В обычном режиме — частота обновления самого оверлея (FPS анимации)
        
    - В полноэкранном режиме игры — частота обновления монитора (Гц)
        
- **Прозрачный фон** и режим «кликсквозь» — не мешает работе с другими окнами
    
- **Автоматическое определение полноэкранного режима** и смены активного окна
    
- **Цветовая индикация** нагрузки (белый → жёлтый → оранжевый → красный)
    

## Системные требования

- **ОС:** Windows (используются WinAPI-функции)
    
- **Python:** 3.7 или новее
    
- **Библиотеки:** см. раздел «Установка»
    
- **GPU:** поддержка NVIDIA (опционально; для других GPU данные не отображаются)
    

## Установка

1. Клонируйте репозиторий или сохраните файл `SystemTracker.py`.
    
2. Установите зависимости через pip:
    
```bash
pip install psutil pynvml
```
    > `tkinter` обычно входит в стандартную поставку Python. Если его нет, установите `python3-tk` через менеджер пакетов вашей системы.
    
3. Запустите скрипт:
    
```bash
python SystemTracker.py
```


## Использование

После запуска в левом верхнем углу экрана появится небольшое окно с информацией:


```text
Active: Chrome
FPS: 60.0
CPU:   12.5%
GPU:   34.2%
RAM:   45.0%
```

- Окно нельзя переместить мышью (это особенность режима «кликсквозь»). Чтобы переместить или закрыть оверлей, завершите процесс Python через Диспетчер задач или консоль (`Ctrl+C` в терминале).

- При переключении на полноэкранное приложение (игра, видео) в строке `FPS` начнёт отображаться частота обновления монитора (например, 144.0).


## Настройка

В начале файла `SystemTracker.py` находятся параметры, которые можно изменить:

|Переменная|Значение по умолчанию|Описание|
|---|---|---|
|`REFRESH_INTERVAL_MS`|1000|Интервал сбора метрик CPU/RAM/GPU (мс)|
|`GUI_UPDATE_INTERVAL_MS`|33|Интервал обновления интерфейса (~30 FPS)|
|`CPU_SAMPLE_INTERVAL`|1|Время замера загрузки CPU (сек)|

Также можно изменить цвета, шрифты, геометрию окна (строка `self.root.geometry("240x125+10+10")` — ширина x высота + смещение по X + смещение по Y).

## Особенности работы

### Обнаружение полноэкранного режима

Окно считается полноэкранным, если его размеры совпадают с разрешением экрана **или** оно не имеет рамки (borderless). Это корректно определяет большинство современных игр и видеоплееров.

### FPS-индикация

- **Обычный режим:** отображается реальная частота кадров анимации самого оверлея (полезно для оценки плавности работы GUI).
    
- **Полноэкранный режим:** отображается текущая частота обновления подключённого монитора (через WinAPI `GetDeviceCaps`). Это позволяет видеть, на какой частоте работает дисплей в игре.
    

### GPU (NVIDIA)

Для работы с GPU используется библиотека `pynvml`. Если видеокарта не NVIDIA или драйвер не поддерживает NVML, в строке GPU будет отображаться `N/A`.

### Прозрачность и кликсквозь

Окно не получает фокус и не перехватывает мышь — можно свободно кликать по любому приложению, даже если оверлей находится поверх него.

## Структура кода

- **Настройки констант** — интервалы обновления и параметры NVML.
    
- **Функции WinAPI**:
    
    - `set_clickthrough()` — делает окно прозрачным для кликов.
        
    - `is_fullscreen_window()` — определяет полноэкранный режим окна.
        
    - `get_current_refresh_rate()` — получает частоту обновления монитора.
        
    - `get_active_window_info()` — возвращает HWND и имя процесса активного окна.
        
- **Класс `Overlay`**:
    
    - Инициализация Tkinter-окна с атрибутами прозрачности и «поверх всех».
        
    - Фоновые потоки: `collect_metrics()` (сбор CPU/RAM/GPU) и `monitor_active_window()` (отслеживание активного окна и полноэкранного режима).
        
    - `update_gui()` — обновляет текст и цвет меток с заданной частотой.
        
- **Запуск приложения** — проверка платформы и создание экземпляра `Overlay`.
